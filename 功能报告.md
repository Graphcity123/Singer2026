## 一、系统概述

基于 Django 6.0 构建的全栈 Web 应用，自动爬取网易云音乐数据，提供歌曲/歌手浏览、搜索、评论、收藏、用户系统等社交功能。前端使用 Django Templates + 原生 CSS + SVG 图标，数据库使用 SQLite3。

> 本项目为课程基础版本的扩展版，基础版代码位于 [task_singers](https://github.com/Graphcity123/task_singers)。本版本在基础版之上通过 vibe coding 新增了用户系统、收藏、点赞、管理员面板等功能。

## 三、功能模块

### 3.1 歌曲列表页 `/songlist`

每日随机排序（`date.today()` 种子 + 预生成映射表），双栏网格布局。每页 20 首，全局连续编号，完整分页组件。

### 3.2 歌手列表页 `/singerlist`

共用随机映射表，五栏网格，每页 25 位。正方形封面图 + 居中歌手名，hover 浮起阴影。

### 3.3 歌曲详情页 `/song/<id>`

320×320 封面大图 + 96px 歌名 + 网易云 iframe 播放器。多栏歌词自适应（`ceil(行数/30)`，最多 5 栏），CSS `column-rule` 栏间分隔。底部固定毛玻璃评论输入栏。

### 3.4 歌手详情页 `/singer/<id>`

艺人封面 + 简介 + 全部作品双栏卡片列表。可收藏/取消收藏歌手。

### 3.5 搜索页 `/search`

四级优先级：歌手名（精确匹配）> 歌手简介 > 歌曲名 > 歌词。Django ORM `Case/When` 标注优先级，毫秒级计时。Tab 切换歌手/歌曲结果。

### 3.6 用户系统

| 功能 | 路由 | 说明 |
|------|------|------|
| 注册 | `/register/` | 用户名 + 密码 + 确认密码，Django `User.objects.create_user` |
| 登录 | `/login/` | Django `authenticate` + `login` |
| 登出 | `/logout/` | Django `logout`，重定向至首页 |
| 个人主页 | `/profile/` | 头像、简介、收藏统计，tab 切换展示收藏的歌曲/歌手/历史评论 |
| 编辑资料 | `/profile/edit/` | 上传头像（`ImageField`）、编辑个人简介 |
| 管理员面板 | `/manage/users/` | 全站用户活跃统计（收藏数/评论数），可查看任意用户的收藏和评论 |

### 3.7 社交功能

| 功能 | 路由 | 说明 |
|------|------|------|
| 收藏歌曲 | `POST /song/<id>/favorite` | 切换收藏状态，`unique_together=(user, song)` |
| 收藏歌手 | `POST /singer/<id>/favorite` | 同上，`unique_together=(user, singer)` |
| 点赞评论 | `POST /comment/<id>/like` | 切换点赞，`unique_together=(user, comment)`，SVG 图标 + CSS mask 染色 |
| 删除评论 | `POST /comment/<id>/delete` | 作者或管理员可删 |

### 3.8 评论系统

- 匿名发表：登录用户可选择匿名（管理员可追踪 `user` 字段）
- 按热度排序：`like_count` 降序 + 时间降序
- 翻页展示：每页 15 条
- 底部固定毛玻璃输入栏（`backdrop-filter: blur`）

## 四、数据模型

```python
class Singer(models.Model):
    id          = IntegerField(primary_key=True)
    name        = CharField(max_length=200)
    image_url   = CharField(max_length=200)
    desc        = TextField(blank=True, null=True)
    source_url  = URLField(max_length=200)
    sort_order  = IntegerField(default=0)          # 预留排序字段

class Song(models.Model):
    id          = IntegerField(primary_key=True)
    name        = CharField(max_length=200)
    image_url   = CharField(max_length=200)
    source_url  = URLField(max_length=200)
    lyrics      = TextField(blank=True, null=True)
    singers     = ManyToManyField(Singer, related_name="songs")
    sort_order  = IntegerField(default=0)

class Comment(models.Model):
    user        = ForeignKey(User, null=True, blank=True, on_delete=SET_NULL)
    name        = CharField(max_length=200)         # 显示昵称（可为匿名）
    content     = TextField()
    song        = ForeignKey(Song, related_name="comments", on_delete=CASCADE)
    create_time = DateField()
    like_count  = IntegerField(default=0)           # 冗余计数，避免 JOIN

class CommentLike(models.Model):                    # 点赞记录
    user        = ForeignKey(User, on_delete=CASCADE)
    comment     = ForeignKey(Comment, on_delete=CASCADE, related_name="likes")
    class Meta: unique_together = ('user', 'comment')

class FavoriteSong(models.Model):                   # 歌曲收藏
    user        = ForeignKey(User, on_delete=CASCADE)
    song        = ForeignKey(Song, on_delete=CASCADE, related_name="favorited_by")
    class Meta: unique_together = ('user', 'song')

class FavoriteSinger(models.Model):                 # 歌手收藏
    user        = ForeignKey(User, on_delete=CASCADE)
    singer      = ForeignKey(Singer, on_delete=CASCADE, related_name="favorited_by")
    class Meta: unique_together = ('user', 'singer')

class UserProfile(models.Model):                    # 用户扩展资料
    user        = OneToOneField(User, on_delete=CASCADE, related_name="profile")
    avatar      = ImageField(upload_to="avatars/", blank=True, null=True)
    bio         = TextField(blank=True, null=True)
```

共 8 个模型（含 Django 自带的 `User`），5 个自定义模型，3 个 `unique_together` 约束防止重复操作。

songs = sorted(songs, key=lambda x: RANDOM_LIST[x.id])

### 5.2 搜索四级优先级

```python
priority = Case(
    When(name__iexact=query, then=Value(0)),
    When(name__icontains=query, then=Value(1)),
    When(desc__icontains=query, then=Value(2)),
    default=Value(3), output_field=IntegerField()
)
```

## 六、爬虫工具

`crawl_hot_songs.py` — 单文件增量爬虫，约 400 行。

### 6.1 核心技术

| 技术 | 用途 |
|------|------|
| `concurrent.futures.ThreadPoolExecutor` | 多线程并发下载（歌手信息、歌词、图片） |
| `requests.Session` | 复用 TCP 连接，Cookie 持久化 |
| `fake_useragent` | 随机 UA 伪装 |
| `json` | 网易云 API 响应解析 |
| `os.makedirs` + `open('wb')` | 图片下载保存 |
| `pickle` | checkpoint 序列化（断点续跑） |

### 6.2 特性

- **增量爬取**：自动跳过已存在的歌曲和歌手（检查数据库）
- **Checkpoint 续跑**：中断后从断点继续，不重复工作（pickle 序列化状态）
- **多线程并发**：`ThreadPoolExecutor(max_workers=8)` 并行处理
- **自动去重**：基于网易云歌曲 ID 检测重复
- **按歌手/专辑爬**：支持 `list.txt` 指定目标歌手 ID 列表
- **配置项**：`consts.py` 定义 `LEAST_SONG`、`LEAST_SINGER` 阈值

### 6.3 反爬策略

| 策略 | 实现 |
|------|------|
| 随机 UA | `fake_useragent` 每次请求随机 |
| Cookie 认证 | 环境变量 `NETEASE_COOKIES` |
| 请求间隔 | `time.sleep(random.uniform(1, 2))` |
| 失败重试 | 最多 5 次，间隔 3~8 秒 |
| Referer 伪装 | `https://music.163.com/` |
